Quick start New in 1.3

createApp is the unidirectional bootstrap: one call wires renderer, scene, camera, lighting, orbit and the loop. Input writes state via setState; modules read state in update() and write the scene — never the reverse. Pass a fixed clock and a seed and the whole sim is deterministic. Copy bootstrap.html as your starting file.

import { createApp, createClock, createEmitter } from '@tuomashatakka/threejs-scenes'

const app = createApp({
  canvas,
  seed:  7,
  clock: createClock({ mode: 'fixed' }),        // deterministic sim steps
  state: { hue: 0.52 },                         // serializable app state
  modules: [{
    name: 'crystal',
    build (ctx)  { /* create objects once; ctx = scene, camera, rng, loop */ },
    update (state, frame) { /* project state onto the scene each tick */ },
    dispose ()   { /* free what build created */ },
  }],
})
canvas.addEventListener('pointerup', () => app.setState({ hue: Math.random() }))
app.start()

particles v2 in 1.3

Particle systems v2: emitter shapes, rate + burst emission, colour/alpha/size curves over lifetime, and a GPGPU path for six-figure counts. Deterministic by construction — every spawn derives from a hash of (seed, slot, cycle), so the same seed and tick sequence reproduce identical particles.

Live demo — particles.html

const fx = createEmitter({
  capacity: 20000, rate: 6000,
  shape:    { kind: 'cone', angle: 0.35, radius: 0.3 },
  speed:    [4, 7], gravity: [0, -6, 0], lifetime: [1, 3],
  color:      [[0, '#fff3b0'], [0.4, '#ff9fd0'], [1, '#7733ff']],
  alphaCurve: [[0, 0], [0.08, 1], [0.7, 1], [1, 0]],
  sizeCurve:  [[0, 0.2], [0.15, 1], [1, 0.4]],
  seed: 7,
})
scene.add(fx.object)
loop.onFrame(ctx => fx.tick(ctx))
button.onclick = () => fx.burst(500)
Exports
ExportSummary
createEmitterCPU-simmed, GPU-billboarded emitter: point/sphere/box/cone/disc shapes, rate + burst(), curves over lifetime, sprite spin, additive/normal blending.
createGpuEmitterGPGPU steady-state field (position/velocity in float render targets) for >50k particles; falls back to the CPU emitter without float-texture support.
sampleCurve / bakeCurve / bakeCurveTextureOver-lifetime curve authoring: [t, value] stops baked to LUTs sampled in-shader.
createParticleEmitterDeprecated v1 emitter — kept for back-compat; prefer createEmitter.

post (WebGL) Expanded in 1.3

An EffectComposer factory, a reorderable named-pass pipeline, and a pass library. The canonical chain is RenderPass → UnrealBloomPass → grade → OutputPass; tone-map exactly once, at the OutputPass.

Live demo — effects.html · post-processing.html · glitch.html · god-rays.html · dof.html

const pipeline = createPostPipeline({ renderer, scene, camera, width, height })
pipeline.register('bloom', new UnrealBloomPass(res, 0.7, 0.4, 0.85))
pipeline.register('crt', createCrtPass({ curvature: 0.12 }))
pipeline.setOrder(settings.pipelineOrder)     // a plain string[] — serialize it
pipeline.setEnabled(settings.enabledPasses)   // { bloom: true, crt: false }
loop.onFrame(ctx => pipeline.render(ctx.delta))
Exports
ExportSummary
createPostPipeline 1.3Named-pass registry over EffectComposer: setOrder(string[]) + setEnabled(flags) driven by serializable config; RenderPass first, OutputPass last, resize forwarded.
createComposerEffectComposer wired with a DepthTexture so DOF / god-rays / soft particles can sample scene depth.
createGradePassLinear-HDR colour grade: tint, contrast, saturation, vignette, grain, corner chromatic aberration.
createGodRaysPass / createDofPass / createFilmGrainPassScreen-space light shafts; CoC depth-of-field + CA; procedural grain.
glitch passes / createHudBeamTransition / createStereoRendererStackable datamosh, a beam reveal, and anaglyph / side-by-side stereo.
createCrtPass / crtCorrectPointer 1.3CRT curvature + vignette + glitch tears, with the inverse pointer warp for distortion-correct picking.
createLensingPass 1.3Screen-space gravitational lens with an event-horizon core; aim it with projectToScreenUv.
createBurnInPass 1.3Phosphor persistence: max-blended decaying history so bright pixels leave saturated trails.

jsx New in 1.1

A higher-level declarative layer over the library. Author scenes as elements; render() mounts them and drives reactivity from the main frame loop — function-valued props are accessors re-read every frame, so reconciliation is the render tick. Works as JSX or, with no build step, as hyperscript (h). Import from @tuomashatakka/threejs-scenes/jsx.

Live demo — jsx-scene.html

Exports & elements
ExportSummary
renderBootstrap + mount a tree onto a canvas; returns { scene, renderer, loop, getCamera, dispose }.
h / jsx / jsxs / FragmentElement constructors — hyperscript and the transpile-target runtime.
signal / derivedMinimal reactivity; accessors are re-evaluated by the loop.
